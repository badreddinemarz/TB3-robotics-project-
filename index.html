Description du projet

Conception et intégration d'une architecture complète de navigation autonome pour un robot mobile TurtleBot3 simulé sous Gazebo, développée en C++ sur ROS 2 Jazzy. Le projet couvre l'ensemble de la chaîne : Perception → Localisation → Planification → Contrôle.

🌐
Environnement de simulation réaliste au format SDF avec obstacles dynamiques
🤖
Intégration TurtleBot3 avec description URDF complète
🔁
Communication bidirectionnelle Gazebo ↔ ROS 2
📡
Exploitation des données LiDAR pour la perception
🗺️
Intégration de SLAM Toolbox pour la cartographie en ligne
🧭
Configuration de Nav2 pour une navigation robuste et globale

Architecture & Étapes clés

1
Étape 01
Conception de l'environnement de simulation
Création d'un fichier world.sdf personnalisé intégrant des obstacles dynamiques. Intégration via le launch file : spawn_warehouse.launch.py
2
Étape 02
Intégration du robot TurtleBot3
Utilisation du modèle SDF depuis turtlebot3_gazebo, chargement de la description URDF, lancement via single_robot.launch.py. Le robot est correctement simulé dans Gazebo à cette étape.
↔
Bridge
Bridge Gazebo ↔ ROS 2
Communication entre simulation et ROS 2 via les topics principaux : /cmd_vel  /odom  /scan
⚠
Problème rencontré : LiDAR non fonctionnel Le topic /scan ne publiait aucune donnée. Cause : absence des plugins nécessaires dans world.sdf.
✓
Solution Ajout des plugins Gazebo pour l'activation du capteur LiDAR. Chaîne de perception finalisée.
// 04

Développement des nodes C++

Navigation basique
/odom /cmd_vel
Exploitation des données d'odométrie, commande du robot et navigation vers des positions prédéfinies.
Évitement d'obstacles
/scan /cmd_vel
Implémentation d'une logique d'évitement réactif basée sur les données LiDAR. Fonctionnel mais sans planification globale.
